# Site Keyword Research Skill

输入一个网站域名，输出完整的关键词研究与竞争度分析报告。

---

## 核心方法论（三大阶段）

```
阶段一：海量搜罗（30+词）
    → 基于主题扩展，收集30+个候选长尾词

阶段二：分层 + 详析（10词）
    → 分层筛选，选出10个词做SERP详细分析

阶段三：定方向（3词）
    → 给出最重要的3个词 + 具体操作建议
```

---

## 输入解析

接受三种格式：
- `https://example.com` → 提取 domain
- `example.com` → 直接使用
- `www.example.com` → 提取 domain

---

## 第一阶段：海量搜罗（30+词）

### 1.1 站点主题分析

使用 `web_fetch` 抓取首页（maxChars: 2000），识别：
- 站点类型（产品工具/内容媒体/电商/SAAS/论坛）
- 核心主题（2-4个主题词）
- 目标用户群体
- 主要功能/服务

同时抓取首页 HTML 标题、meta description。

### 1.2 种子词扩展（目标：30+词）

围绕核心主题，用 `browser` 搜索以下来源，系统性地扩展关键词列表：

**来源 1：Google PASF（最重要）**
搜索每个核心主题词，从 "People Also Search For" 区域提取长尾

**来源 2：相关搜索扩展**
- `核心主题 + tool`
- `核心主题 + free`
- `核心主题 + online`
- `核心主题 + software`
- `核心主题 + alternative`
- `核心主题 + vs`
- `核心主题 + examples`
- `核心主题 + generator`
- `核心主题 + maker`
- `核心主题 + creator`

**来源 3：问题型长尾**
- `how to 核心主题`
- `what is 核心主题`
- `why 核心主题`
- `核心主题 + questions`
- `核心主题 + tutorial`
- `核心主题 + guide`

**来源 4：竞品联想**
- 搜索 `best 核心主题 tools`
- 搜索 `核心主题 alternatives`
- 从结果页提取竞品相关的长尾词

**收集要求：**
- 最终候选词库 ≥ 30个
- 每个词记录：原始词、搜索变体、关键词类型、搜索意图

---

## 第二阶段：关键词分层 + 10词详析

### 2.1 关键词分层（四层模型）

根据搜索量和竞争度，将30+词分为四层：

| 层级 | 定义 | 数量 | 策略 |
|------|------|------|------|
| **核心词** | 高搜索量，主赛道 | 1-2个 | 长期目标，品牌期做 |
| **中尾词** | 中等搜索量，明确意图 | 3-5个 | 3-6个月内容建设 |
| **长尾词** | 低搜索量，精准需求 | 10-15个 | 立即行动，快速见效 |
| **问题词** | 问句形式，信息需求 | 5-10个 | 博客内容，覆盖漏斗顶端 |

### 2.2 筛选标准（30词 → 10词）

**优先保留：**
- 搜索意图明确（Transactional / Commercial Investigation）
- 竞品内容薄弱的词（有页面但内容浅)
- 包含高意图修饰词（free、best、generator、maker、alternative）
- 与站点主题高度相关

**优先剔除：**
- 过于泛的词（如只用 "software"）
- 与站点无关的词
- 无搜索信号的词

**最终选10个词**，覆盖各层级，确保多样性。

### 2.3 SERP 详细分析（10词）

对每个入选词，调用 `browser` 打开 Google：

```
https://www.google.com/search?q=<编码关键词>&hl=en
```

用 `snapshot compact=true` 抓取，记录：

| 分析维度 | 记录内容 |
|---------|---------|
| 广告主数量 | 0 / 1-3 / 4-6 / 6+ |
| 广告主类型 | （列出主要广告主） |
| Featured Snippet | 有 / 无 |
| 视频结果 | 有 / 无（数量） |
| 前10域名 + 类型 | 工具/博客/目录/论坛/官方 |
| 前10内容深度 | 薄页(<200字) / 中等 / 深度(1000+字) |
| 高权重站数量 | Wikipedia/Quora/Amazon/大型媒体 |
| PASF 推荐词 | 列出Google推荐的关联搜索词 |

**竞争度打分（1-5分，分低=机会大）：**

| 维度 | 1分 | 2分 | 3分 | 4分 | 5分 |
|------|-----|-----|-----|-----|-----|
| 广告主 | 0 | 1-2 | 3-5 | 6-8 | 8个以上 |
| 高权重站 | 0 | 1 | 2-3 | 4-5 | 5个以上 |
| Featured Snippet | 无 | — | 1个 | — | 2个以上 |
| 视频结果 | 无 | — | 1-2个 | — | 3个以上 |
| 内容深度 | 全薄页 | 多数薄 | 混合 | 多数深 | 全深度 |

**总分 5-10 → 🟢 低竞争（立即行动）**
**总分 11-17 → 🟡 中竞争（可切入）**
**总分 18-25 → 🔴 高竞争（观望/迂回）**

---

## 第三阶段：定方向（最重要3词）

### 3.1 最终3词选择

综合以下标准，选出最重要的3个词：

1. **竞争度低**（分数 ≤ 10）
2. **与站点主题高度匹配**
3. **搜索意图清晰且可转化**
4. **现有结果内容薄弱**（有机会碾压）

### 3.2 关键词策略报告

对每个 Top 3 词，给出：

```
关键词：{词}

分层：{核心词/中尾词/长尾词/问题词}

竞争度：{分数}分 / {低中高三档}

机会描述：
{一句话说明为什么这个机会现在存在}

推荐动作（具体到操作）：
{明确告诉用户第一步做什么、第二步做什么}

落地页建议：
- 标题：{建议标题}
- 核心卖点：{3个核心卖点}
- CTA：{建议的Call to Action}
- 覆盖长尾：{建议覆盖的PASF长尾词}

外链机会：
{对应的外链建设策略}
```

## 输出报告结构

按以下章节输出完整报告：

```
一、站点定位分析
    1.1 站点基础信息
    1.2 核心主题提取

二、关键词分层矩阵
    30+词完整列表（按层级分组）

三、10词SERP详细分析
    每个词：打分表 + 竞争者快照 + PASF机会

四、综合机会矩阵
    所有分析词的横向比较表

五、Top 3 关键词定方向（核心章节）
    每个词的：竞争度 + 机会描述 + 具体操作建议 + 落地页要素

六、外链策略建议
    基于Top 3词的外链机会识别

七、数据说明与后续建议
```

## 关键原则

- **严禁跳过30词阶段**：没有足够样本无法做分层判断
- **10词分析必须打分**：不打分就无法横向比较优先级
- **3词必须有具体操作建议**：不能只给词不给动作
- **用 Browser 抓 Google**：SERP数据是核心，DuckDuckGo仅供参考
- **PASF 词必须提取**：Google推荐的关联搜索是免费的高价值洞察

---

## 错误处理

| 场景 | 处理 |
|------|------|
| web_fetch 403/404 | 跳过，用已知主题信息继续分析 |
| 站点是全新站（DA≈0）| 标注"新站，竞品内容质量差距是核心机会指标" |
| Google 弹出验证 | 改用 web_search 补充，报告中标注数据来源 |
| 候选词不足30个 | 降低门槛至20个，说明原因，继续分析 |
